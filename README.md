# 📋 PROJETO MECÂNICA — ERP para Oficinas

Este repositório contém os documentos de concepção e planejamento ágil de um sistema ERP voltado para oficinas mecânicas, desenvolvido na disciplina **Inovação, Tecnologias e Futurismo** (Prof. José Luiz Lanzarini).

---

## 📁 Arquivos do Projeto

### 1. `Documento_inicial_projeto_software.pdf`

Documento de concepção inicial do projeto. Serve como ponto de partida formal, estruturado nos seguintes tópicos:

- **Identificação do Projeto** — nome, equipe e data
- **Problema a Ser Resolvido** — desorganização nos processos de atendimento, controle de ordens de serviço e gestão de estoque em mecânicas
- **Objetivo do Sistema** — centralizar todas as rotinas da oficina em um único sistema
- **Público-Alvo** — donos de mecânicas, mecânicos, atendentes e profissionais automotivos autônomos
- **Descrição da Solução** — sistema ERP completo com agendamentos, gestão de OS, controle de estoque e integração com pagamentos
- **Funcionalidades Principais** — listagem das entregas do sistema
- **Diferencial/Inovação** — comparação com soluções existentes
- **Tecnologias Utilizadas** — linguagens, banco de dados e frameworks
- **Viabilidade** — estimativa de tempo e recursos necessários
- **Impacto Esperado** — impacto para usuários e para o negócio

---

### 2. `backlog_produto.pdf`

Documento de Product Backlog estruturado em formato de User Stories. Contém:

- **Objetivo** — organizar e priorizar todas as funcionalidades do sistema
- **Estrutura do Backlog** — tabela com ID, User Story, Prioridade, Valor, Complexidade e Responsável
- **Formato das US** — "Como [usuário], quero [funcionalidade], para [benefício]"
- **Critérios de Priorização** — Alta/Média/Baixa prioridade, valor para o usuário e esforço de desenvolvimento

---

### 3. `Atividade Agil Estruturada.pdf`

Documento principal da atividade prática de gestão ágil. Reúne todo o planejamento do projeto em 9 seções:

| Seção | Conteúdo |
| --- | --- |
| **1. Product Discovery** | Problema, solução, público-alvo e valor principal |
| **2. User Stories** | 10 histórias de usuário com valor e esforço mapeados |
| **3. Validação INVEST** | Checagem de qualidade das US (Independente, Negociável, Valiosa, Estimável, Pequena, Testável) |
| **4. Story Points** | Estimativas de esforço com justificativas para cada US |
| **5. MoSCoW** | Priorização em Must Have, Should Have, Could Have e Won't Have |
| **6. WSJF** | Priorização por valor x esforço (Weighted Shortest Job First) |
| **7. MVP** | Escopo mínimo viável: cadastro, OS, estoque e orçamentos |
| **8. Release Planning** | 4 sprints planejados com entregas definidas |
| **9. Análise de Riscos** | Principais riscos identificados, impacto e mitigação |

## 🚀 Sobre o Sistema

O sistema proposto é um **ERP completo para oficinas mecânicas**, com foco em:

- Cadastro de clientes e veículos
- Registro e gestão de Ordens de Serviço (OS)
- Controle de estoque de peças em tempo real
- Emissão de orçamentos digitais com aprovação via WhatsApp
- Agendamento online pelo aplicativo
- Integração com pagamentos (PIX e cartão)
- Relatórios financeiros e histórico de reparos
